# cms-for-lofi-focus-ios\n\nStrapi v4 project that powers mentor personas and insights for the Lofi Focus apps.\n\n## Getting started\n\n1. Copy the environment template and fill in secrets:\n   `ash\n   cp .env.example .env\n   `\n2. (Optional) point DATABASE_URL at Neon if you want to test against the real Postgres instance; otherwise the default SQLite file .tmp/data.db is used.\n3. Install dependencies and run the dev server:\n   `ash\n   npm install\n   npm run develop\n   `\n4. Open [http://localhost:1337/admin](http://localhost:1337/admin) to create the first admin account.\n\n### Production-style build\n\n`ash\n# assumes .env has DATABASE_URL pointing at Neon (direct string)\nnpm install\nnpm run build\nNODE_ENV=production npm run strapi migrations:run\nNODE_ENV=production npm run start\n`\n\n## Content architecture\n\n- **Mentor Group** (mentor-group)\n  - Fields: 	itle, slug, description, order, domain, personaSlug, mediaMasterUrl, mediaFallback720Url, mediaPosterUrl.\n  - Relation: one group has many mentor-insight entries.\n- **Mentor Insight** (mentor-insight)\n  - Fields: slug, order, mainIdea, shortExplanation, callToAction.\n  - Relation: many insights belong to a single mentor group.\n\nBoth REST endpoints (/api/mentor-groups, /api/mentor-insights) are public for GET requests so the client apps can fetch without authentication.\n\n## Useful scripts\n\n- 
pm run develop – start Strapi in watch mode.\n- 
pm run build – build the Strapi admin panel.\n- 
pm run start – start Strapi in production mode.\n- 
pm run strapi migrations:run – manually run pending database migrations.\n\nSee docs/DEPLOYMENT_RENDER_NEON.md for Render + Neon deployment instructions and STRAPI_CDN_REQUIREMENTS.md for CDN expectations.\n
